Chapter 1: Meet Your First Program

Welcome to the world of coding! Every coder begins with the classic phrase:

print("Hello, World!")

This little line is your first program. What does it do? It tells your computer to say Hello, World! in its own language.

How to Try It:

  1. Open a website like Replit or install Python on your computer.
  2. Type the above code and press run.
  3. Ta-da! You've just written your first program.

AI Prompt Practice:

  • "Explain what print("Hello, World!") does in Python."
  • "Can you give me another example of how to use print?"

See how your question shapes the answer? A clear and specific prompt gets you the best help.

Chapter 2: Speak to Your Machine

Coding is like speaking to your computer, and prompting is like asking an intelligent assistant to think with you. Both skills require you to be clear and specific.

What is Prompt Engineering?

Prompt engineering is the art of framing your questions or instructions to get the best responses from AI tools. Think of it like programming, but instead of code, you use natural language.

Examples:

  • A vague prompt: "Tell me something about Python."
  • A specific prompt: "What are the main features of Python, and why is it great for beginners?"

The more details you provide, the better the response will be.

Your First Prompt Exercise

Ask an AI assistant like ChatGPT: "Explain Python in simple words for a 10-year-old." Notice how it adapts its response to match the audience.

Combining Coding and Prompts

You can also ask AI to help with your code! For example: "Write a Python program that calculates the area of a circle." The AI will generate a program, and you can learn from it.

Chapter 3: Variables and Values

A variable in programming is like a box where you store information. Imagine labeling a box "My Favorite Number" and putting the number 7 inside.

favorite_number = 7 print(f"My favorite number is {favorite_number}")

Why Are Variables Important?

  • They let your programs remember things, like a player's score in a game or a user's name.
  • They make your code more flexible and powerful.

Prompting Practice

  • "Explain how variables work in Python with examples."
  • "How do I create a variable for storing my age?"

Notice how clear questions lead to useful examples.

Chapter 4: Decisions and Loops

Programs often need to make decisions. For example, imagine you're coding a game where the player wins if their score is 10 or more.

score = 12 if score >= 10: print("You win!") else: print("Keep trying!")

Loops: Doing Things Again and Again

for i in range(5): print("This is loop number", i)

Prompting Practice

  • Ask an AI: "How do if-else statements work in Python?"
  • Challenge it: "Write a loop that counts from 1 to 10."

Chapter 5: Debugging and Asking Why

Even the best programmers make mistakes, but debugging helps you fix them. Debugging is like solving a mystery: you find clues (error messages) and figure out what went wrong.

Debugging Tips

  1. Read the error message carefully.
  2. Check your code for typos.
  3. Test small parts of your program to find the problem.

Prompting Practice

  • "Why am I getting a SyntaxError in this code?"
  • "What's the best way to debug a Python program?"

Chapter 6: Build a Mini Project

Let's create something fun: a simple calculator!

def add_numbers(a, b): return a + b num1 = int(input("Enter the first number: ")) num2 = int(input("Enter the second number: ")) result = add_numbers(num1, num2) print("The result is:", result)

Prompting Practice

  • "How can I make this calculator handle more operations, like subtraction?"
  • "Can you help me make this code look more professional?"

Chapter 7: Thinking Like a Developer

Being a developer isn't just about writing code; it's about solving problems. Break problems into smaller steps, test your ideas, and keep improving.

Prompting Practice

  • Use AI tools as your partner. Ask things like:
  • "What are the steps to build a to-do list app?"
  • "How do I make my program run faster?"
